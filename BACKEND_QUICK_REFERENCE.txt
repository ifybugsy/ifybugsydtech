================================================================================
                            QUICK START (5 MIN)
================================================================================

1. INSTALL
   $ cd backend
   $ npm install

2. CONFIGURE
   $ cp .env.example .env
   (Edit .env with MongoDB URI and JWT secret)

3. START
   $ npm run dev
   (Server runs on http://localhost:5000)

4. TEST
   $ curl http://localhost:5000/health
   (Should return: {"status":"OK",...})

================================================================================
                            ENVIRONMENT SETUP
================================================================================

REQUIRED:
   NODE_ENV=development
   PORT=5000
   CLIENT_URL=http://localhost:3000
   MONGODB_URI=mongodb://localhost:27017/ifybugsy
   JWT_SECRET=your-secret-key

OPTIONAL:
   REDIS_HOST=localhost
   REDIS_PORT=6379
   STRIPE_SECRET_KEY=...
   SMTP_HOST=smtp.gmail.com

See backend/.env.example for all options

================================================================================
                            API ENDPOINT SUMMARY
================================================================================

AUTHENTICATION
   POST   /api/auth/register           - Register user
   POST   /api/auth/login              - Login user
   GET    /api/auth/me                 - Get current user
   PUT    /api/auth/profile            - Update profile

COURSES
   GET    /api/courses                 - List all courses
   GET    /api/courses/:id             - Get course details
   POST   /api/courses                 - Create course
   PUT    /api/courses/:id             - Update course
   DELETE /api/courses/:id             - Delete course
   POST   /api/courses/:id/enroll      - Enroll in course

PRODUCTS
   GET    /api/products                - List all products
   GET    /api/products/:id            - Get product details
   POST   /api/products                - Create product
   PUT    /api/products/:id            - Update product
   DELETE /api/products/:id            - Delete product
   PUT    /api/products/:id/stock      - Update stock

ORDERS
   POST   /api/orders                  - Create order
   GET    /api/orders/my-orders        - Get user orders
   GET    /api/orders/:id              - Get order details
   PUT    /api/orders/:id/status       - Update status
   GET    /api/orders                  - List all (admin)

PAYMENTS
   POST   /api/payments                - Create payment
   GET    /api/payments/my-payments    - Get user payments
   GET    /api/payments                - List all (admin)
   GET    /api/payments/stats          - Get stats (admin)

COMMUNITY
   GET    /api/community               - List posts
   GET    /api/community/:id           - Get post
   POST   /api/community               - Create post
   POST   /api/community/:id/reply     - Add reply
   POST   /api/community/:id/like      - Like post
   PUT    /api/community/:id/mark-solved - Mark solved
   DELETE /api/community/:id           - Delete post
   POST   /api/community/:id/flag      - Flag post

BLOG
   GET    /api/blog                    - List posts
   GET    /api/blog/:id                - Get post
   POST   /api/blog                    - Create post
   PUT    /api/blog/:id                - Update post
   POST   /api/blog/:id/publish        - Publish post
   DELETE /api/blog/:id                - Delete post
   POST   /api/blog/:id/like           - Like post
   POST   /api/blog/:id/comments       - Add comment

USERS
   GET    /api/users                   - List users (admin)
   GET    /api/users/:id               - Get user
   PUT    /api/users/:id/role          - Update role (admin)
   PUT    /api/users/:id/deactivate    - Deactivate (admin)

================================================================================
                            SOCKET.IO EVENTS
================================================================================

EMIT (Client to Server):
   ├─ join-community              - Join community room
   ├─ new-community-post          - Create post
   ├─ new-community-reply         - Add reply
   ├─ join-course                 - Join course room
   ├─ course-progress             - Update progress
   ├─ new-blog-post               - Publish blog
   ├─ new-order                   - Create order
   ├─ payment-completed           - Complete payment
   └─ join-admin                  - Join admin room

LISTEN (Server to Client):
   ├─ user-joined                 - User joined community
   ├─ community-post-created      - New post created
   ├─ community-reply-created     - New reply added
   ├─ user-joined-course          - User joined course
   ├─ progress-updated            - Progress updated
   ├─ blog-post-published         - Blog published
   ├─ order-created               - Order confirmed
   ├─ new-order-admin             - Admin alert
   ├─ payment-success             - Payment success
   ├─ payment-received-admin      - Admin alert
   └─ user-disconnected           - User left
